Machotkey API Reference

Complete reference for the Machotkey Lua scripting API. All modules are exposed under the global system table.

system
Core utilities: key bindings, timing, tasks, and process control.
system.on_keyfunction
Register a global hotkey. The callback fires whenever the key combination is pressed.
ParameterTypeDescription
combostringKey combination string, e.g. "ctrl+shift+a"
options (optional)table{ swallow?: boolean } — if swallow is true, the keypress is not forwarded to other apps
callbackfun()Function called when the combo is triggered
system.on_key(combo, callback)
system.on_key(combo, options, callback)
Example
-- Simple hotkey
system.on_key("cmd+shift+h", function()
    print("triggered!")
end)

-- Swallow the keypress so other apps don't see it
system.on_key("ctrl+f1", { swallow = true }, function()
    system.mouse.click(system.mouse.Button.LEFT)
end)
system.remove_keybindfunction
Remove a previously registered hotkey.
ParameterTypeDescription
combostringThe key combination to unregister
Example
system.on_key("cmd+shift+h", function() print("hello") end)
-- later...
system.remove_keybind("cmd+shift+h")
system.waitfunction
Pause execution for the given duration. If unit is omitted, values are interpreted as seconds.
ParameterTypeDescription
timenumberDuration to wait
unit (optional)"s" | "ms" | "us"Time unit. If omitted, time is in seconds.
system.wait(time)
system.wait(time, unit)
Example
system.wait(0.5)        -- wait half a second (default unit is seconds)
system.wait(250, "ms")  -- wait 250 milliseconds
system.wait(2, "s")     -- wait 2 seconds explicitly
system.wait(100, "us") -- wait 100 microseconds
system.get_timefunction
Returns a monotonic time value from when the runner started tracking. Omitting unit returns seconds (floating point).
ParameterTypeDescription
unit (optional)"s" | "ms" | "us"Time unit for the return value. Defaults to "s" if omitted.
Returns  number
Example
local start = system.get_time("ms")
system.wait(100, "ms")
local elapsed = system.get_time("ms") - start
print("elapsed ms: " .. elapsed)
system.set_timeoutfunction
Schedule a callback to run once after a delay. Uses the same time rules as system.wait (time is in seconds if unit is omitted).
ParameterTypeDescription

timenumberDelay duration (seconds if unit omitted)
unit (optional)"s" | "ms" | "us"Time unit
Returns  integer — timer ID
Example

    print("fired after 1 second")

Returns  integer — timer ID

    print("tick")

Stop a running interval.
ParameterTypeDescription
timer_idintegerID returned by system.set_interval
Example

system.screen.get_next_framefunction
Wait until the shared frame index advances (new frame available). Outside a cooperative context (plain top-level script code) this polls in a tight loop. Inside scheduled callbacks—hotkey handlers, timers, or UI events—the runtime yields between polls and timeout_ms is honored (default 1000 ms).
ParameterTypeDescription
timeout_ms (optional)numberMaximum wait in milliseconds (coroutine/task path only; default 1000)
Returns  Cooperative context: true, or false, "timeout". Plain top-level synchronous calls return only true once a new frame arrives.
Example
system.screen.wait_until_ready()
while true do
  local ok, err = system.screen.get_next_frame(500)
  if not ok then break end -- timeout in coroutine
  local pt = system.screen.find_color(255, 0, 0, 0, { x = 0, y = 0, w = 400, h = 300 })
end
system.screen.find_colorfunction

rect (optional)RectSearch region. Searches entire frame if omitted.
reverse (optional)booleanSearch right-to-left
reverse_vertical (optional)booleanSearch bottom-to-top
Returns  Point|nil

-- fuzzy match (tolerance then rect)

system.screen.ocr
Optical character recognition on screen frames or pixel buffers. Two backends are available: fast (lower accuracy, higher speed) and accurate (higher accuracy, slower).

options (optional)table{ region?: Rect } — limit scan to a sub-region
Returns  OCRVector
Example

  local seg = words[i]

system.opencv
Load images and perform computer vision operations using OpenCV. Returns Mat objects for image processing.
system.opencv.load_from_filefunction
Load an image file into a Mat.
ParameterTypeDescription
filenamestringPath to the image file
Returns  Mat
Example

Mat methods
All image operations return a new Mat. Mats are garbage collected automatically, but call :free() early when working with large images to manage memory pressure.
MethodReturnsDescription
:to_bgr()MatConvert from BGRA to BGR. Errors if the Mat is not 4-channel.
:to_bgra()MatWiden OpenCV storage from CV_8UC3 (BGR) to BGRA (CV_8UC4).
:to_pixelbuffer()PixelBufferConvert to a raw pixel buffer
:resize(w, h)MatResize to exact pixel dimensions
:upsample()MatDouble the image size (2×)
:downsample()MatHalve the image size (0.5×)
:extract_channel(channel)MatExtract a single color channel. channel is 1-indexed (1 = first channel).
:match_template(template, options?)TemplateMatchVectorFind occurrences of template within this Mat. See options below.
:free()—Release native memory immediately
Mat:match_template options
:match_template(template, nil) uses an internal threshold of -1 to return OpenCV's single best bounding box. Passing { threshold = 0.94 } keeps every localized peak above 0.94. Use raw = true for the brute-force pyramid-free path.
FieldTypeDescription
thresholdnumber (0–1)Minimum score to keep iterating when searching multiple hits. Omit (or explicitly -1) to request the hottest single correlation.
rawbooleanIf true, skips pyramid acceleration and runs a direct match
maskMatOptional mask Mat applied during matching
scene_downsampledMatPre-downsampled scene Mat for pyramid matching
template_downsampledMatPre-downsampled template Mat for pyramid matching
mask_downsampledMatPre-downsampled mask Mat for pyramid matching
TemplateMatchVector
Returned by :match_template. Behaves as an array of match results, supporting # and ipairs.
FieldTypeDescription
[i]match1-based access to each match result
[i].xintegerX coordinate of the match in the scene
[i].yintegerY coordinate of the match in the scene
[i].scorenumberMatch confidence score (0–1)
End-to-end
local scene = system.opencv.load_from_file("screens/last.png")
local tpl = system.opencv.load_from_file("assets/button_template.png"):resize(96, 64)
local matches = scene:match_template(tpl)
if #matches >= 1 then
  local m = matches[1]
  print("hit", m.x, m.y, m.score)
end

scene:match_template(tpl, { threshold = 0.9, raw = false })

manifest.json
Every macro project must include a manifest.json at its root. It declares the entry point, requested permissions, file access rules, and optional runtime guardrails. Only entry is required.
Example
{
  "name": "My Macro",
  "version": "1.0.0",
  "entry": "main.lua",
  "permissions": [
    { "name": "screen_capture" },
    { "name": "mouse_control", "optional": true }
  ],
  "files": [
    "data/config.json",
    { "path": "logs/output.txt", "write": true }
  ],
  "rate_limits": {
    "mouse_events_per_sec": 60,
    "cpu_throttle_percent": 50
  }
}

Permission object
Each entry in the permissions array is an object with the following fields.
FieldTypeRequiredDescription
namestringYesPermission identifier. Known values: "screen_capture", "keyboard_listen", "keyboard_control", "mouse_control"
optionalbooleanNoIf true, a denied permission does not block execution of the macro
File rule
Each entry in files is either a plain path string (grants read access) or an object for explicit access control. At least one of read or write must be true when using the object form.
FieldTypeRequiredDescription
pathstringYesFile path relative to project root
readbooleanConditionalAllow read access to this path
writebooleanConditionalAllow write access to this path
optionalbooleanNoIf true, the permission prompt for this file can be skipped
"data/config.json" ← shorthand, grants read
{ "path": "logs/out.txt", "write": true } ← explicit write access
{ "path": "cache.db", "read": true, "write": true } ← read + write

PixelBufferclass
Raw BGRA pixel data backed by native memory. Obtained from system.screen.buffer, Mat:to_pixelbuffer(), or buffer operations. Mutable buffers support pixel writes; read-only buffers (e.g. the live screen buffer) do not.
